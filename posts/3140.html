虚拟机装完kali之后的配置

kali linux
Publish Date:   2020-04-11
Word Count:   874
Read Times:   3 Min
Read Count:  

我们会进行以下配置：
1.配置VMware tools
2.把kali的apt源换成国内阿里云的源，实现加速。

安装VMware tools

介绍

上一篇文章中我也有提到关于VMware tools的功能，这里我就在啰嗦一遍。第一呢、VMware tools可以把你虚拟机里面系统的界面放大到你想要的尺寸，甚至全屏，这样看着会舒服些。第二就是可以实现你物理机上面的文件和虚拟机上面的文件实现共享，就是你可以从物理机拖动文件到虚拟机。

开始安装

安装的话有两种方法：
第一种就是直接用VMware虚拟机自带的光盘，但这个我搞了好几次都有报错，听朋友说自带的那个兼容性也不是很好。
第二种方法就是通过终端进行安装。我们这里就直接讲第二种方法了。

首先我们先对我们的apt命令进行检查，获取软件包的状态。
打开终端输入以下命令：

apt update

上面的命令完成之后：

apt install open-vm-tools-desktop fuse -y

这一步的包有点多，需要点时间。

完成之后重启一下，重启完之后就可以生效了。

更新kali的源为国内的阿里云的源

介绍

为什么要更新到国内的源呢？就是因为kali自带的apt源是国外的，我们下载软件包的时候回很慢。所以我们配置成国内的源。国内的源有很多，但是还是阿里的源带宽高一点。
当然你也可以用清华的源，csdn，中科大的源一会我会在下面放一些源。你自己选就行了。

开始操作

别人大佬都是在终端通过vim编辑器进行修改的，对我这个小白来说那个我还不太习惯，我们这里就直接通过图形界面进行修改了。

点击位置里面的计算机，你会进入到文件夹目录。
找到etc/apt/sources.list 文件鼠标右击选择文本编辑器打开。

我们可以把他原来第八行的源给他删除，也可以像我这样直接在前面加上一个# （这里是#加一个空格，空格不要忘了），之后再下面粘贴上阿里云的源。反正你就跟我搞成一样的保存就可以了。

检查是否更换成功了

打开终端输入以下命令：

apt update


我们可以看到命中得到是我们刚才更换的aliyun的源。

附上一些国内的源：

#中科大：
deb http://mirrors.ustc.edu.cn/kali kali-rolling main non-free contrib
deb-src http://mirrors.ustc.edu.cn/kali kali-rolling main non-free contrib
阿里云：
deb http://mirrors.aliyun.com/kali kali-rolling main non-free contrib
deb-src http://mirrors.aliyun.com/kali kali-rolling main non-free contrib
清华大学：
deb http://mirrors.tuna.tsinghua.edu.cn/kali kali-rolling main contrib non-free
浙大：
deb http://mirrors.zju.edu.cn/kali kali-rolling main contrib non-free
deb-src http://mirrors.zju.edu.cn/kali kali-rolling main contrib non-free
东软大学：
deb http://mirrors.neusoft.edu.cn/kali kali-rolling/main non-free contrib
deb-src http://mirrors.neusoft.edu.cn/kali kali-rolling/main non-free contrib

补充

通过我上一篇文章和这一篇两篇文章，主要写了在vm中安装kali，为vm安装VMwaretools，以及修改kali的源为国内阿里云的源，这就把kali的基本配置就算完成了。
